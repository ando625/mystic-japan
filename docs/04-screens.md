## 共通レイアウト

### PC

- 左または下部にゲーム風ナビゲーションを配置する。
- メイン領域は星空・オーロラ・霧の背景を広く見せる。
- 情報カードは半透明のガラスUIにし、紫の外光をつける。
- 重要なCTAのみ強く発光させる。

### Mobile

- 参考画像のスマホ画面のように、下部固定ナビゲーションを使う。
- カードは2カラムを基本にし、詳細画面は縦スクロールを許容する。
- ボタン文言は短くし、アイコンはlucide-reactを使う。

## 主要画面詳細

### 1. ホーム

目的:

- 一瞬で「幻想的な和風ファンタジーRPG」と伝える。
- 旅の開始ボタンから地図へ導線を作る。

要素:

- 月明かり、鳥居、夜空、星、霧を感じるフルスクリーン背景
- タイトル「日本神秘紀行」
- サブコピー「神々の記憶を巡る旅」
- CTA「旅を始める」
- 下部ナビゲーション

### 2. 日本地図

目的:

- 日本地図上でスポットを選ぶ探索体験を提供する。

要素:

- React Leaflet地図
- 神秘スポットピン
- ピンホバー時の紫発光
- ピンクリック時の波紋エフェクト
- 選択スポットの小型プレビューカード

クリック演出:

1. ピンをクリック
2. 小型カードが表示
3. 詳細へ進むとカードが中央へズーム
4. 背景暗転
5. 詳細画面がフェードイン

### 3. スポット一覧

目的:

- 図鑑のようにスポットを一覧し、解放状態を把握できる。

要素:

- カテゴリタブ
- スポットカードグリッド
- レア度星表示
- 未解放スポットのロック表示
- 解放済みの紫水晶アクセント

### 4. スポット詳細

目的:

- スポットの魅力とゲーム的な報酬を提示する。

要素:

- 大きなメインビジュアル
- スポット名、都道府県、レア度
- 説明
- ギャラリー / BGM / 動画 / お気に入り風ボタン
- 「この絶景を解放する」CTA
- 解放済みの場合は「解放済み」と表示

### 5. BGM再生

目的:

- スポットの世界観を音楽で楽しむ。

要素:

- 円形ジャケット
- 再生/停止ボタン
- シークバー
- `music_url` がない場合は「BGM準備中」

### 6. 神話・歴史・豆知識

目的:

- 図鑑として読み物体験を提供する。

要素:

- タブ: 神話・伝説 / 歴史 / 豆知識
- テキストカード
- 関連画像

### 7. AI旅ガイド

目的:

- スポット情報を前提に、AIガイドと会話する。

要素:

- ユーザー入力欄
- 送信ボタン
- AI回答カード
- 送信中ローディング
- エラー時の再試行

### 8. 図鑑

目的:

- 解放率と収集進捗を明確にする。

要素:

- 解放率リング
- 神秘ポイント合計
- 解放した数
- すべて / 解放済み / 未解放タブ
- スポットカード

### 9. ミッション・称号

目的:

- 継続プレイの動機を作る。

要素:

- 称号カード
- 進行中 / 達成済み状態
- 進捗数値
- 獲得時の紫光演出

### 10. マイページ

目的:

- ユーザーの旅の進捗をまとめる。

要素:

- ユーザー名、レベル風表示
- 神秘ポイント
- 解放スポット数
- 獲得称号数
- お気に入り風スポット表示
