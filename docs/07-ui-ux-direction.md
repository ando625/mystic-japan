# UI/UXデザイン方針

## 目標

旅行サイトではなく、「和風ファンタジーRPGのメニュー画面」として感じられるUIを作ります。

参考画像の方向性から、以下を最優先にします。

- 夜
- 月明かり
- 星空
- オーロラ
- 紫と青の発光
- 鳥居
- 神社
- 神域
- 霧
- 精霊感
- 高級感のあるゲームUI

## カラーパレット

単色の紫だけに寄せず、深い夜色、青、紫、淡い金を組み合わせます。

| 用途 | 色 |
| --- | --- |
| 背景最深部 | `#050816` |
| 夜空ネイビー | `#08142d` |
| 深い藍 | `#10224a` |
| 神秘パープル | `#7c3aed` |
| 発光バイオレット | `#a855f7` |
| 青白い光 | `#bfdbfe` |
| 星光 | `#e0f2fe` |
| 古金アクセント | `#d6b56d` |
| 危険・未解放 | `#64748b` |

## タイポグラフィ

- 見出しは細めで余白を持たせ、青白い発光を軽く付ける。
- 本文は読みやすさを優先し、白または薄紫を使う。
- 文字の光彩は控えめにし、長文の可読性を落とさない。
- 日本語表示を前提に、フォントはシステムフォントから開始する。

## 背景

背景は複数レイヤーで構成します。

1. 深い夜空グラデーション
2. 星粒
3. オーロラの淡い帯
4. 霧の半透明レイヤー
5. 遠景の鳥居・山影・神社シルエット

Framer Motionで低速のパララックスを付けます。背景は主張しすぎず、UIの判読性を守ります。

## カード

- グラスモーフィズム
- `backdrop-blur`
- 半透明の濃紺
- 1pxの薄紫ボーダー
- hover時に少し拡大
- hover時に紫の外光を強める

カードは情報のまとまりにだけ使い、ページ全体をカードだらけにしません。

## ボタン

### Primary

- 紫の発光
- 細い金または薄紫の縁
- hover時に光が広がる
- active時に少し沈む

### Secondary

- 半透明
- 薄紫ボーダー
- hover時に背景を少し明るくする

## アニメーション

| 対象 | 動き |
| --- | --- |
| ページ表示 | フェードイン + 少し上へ移動 |
| カード | hoverで `scale: 1.03` |
| ボタン | hoverで発光強化 |
| 背景 | ゆっくりしたパララックス |
| ピン | hoverで発光、クリックで波紋 |
| 詳細遷移 | カードズーム + 背景暗転 + フェードイン |
| 御朱印獲得・解放演出 | 中央の結晶発光 + 御朱印獲得 + ポイント加算 |

## クイズ・御朱印UI

- スポット詳細には手動の「この絶景を解放する」ボタンを置かない。
- 「訪問済みにする」ボタンで御朱印を付与しない。
- 御朱印パネルでは、神話クイズで4問中3問以上正解すると御朱印を獲得できることを伝える。
- クイズ画面では正解数を `現在の正解数 / 3` で表示する。
- 4問回答後に3問正解できなかった場合だけ、画面下部に「もう一回」ボタンを表示する。
- 御朱印獲得済みの場合は、リトライ導線を表示しない。

## レスポンシブ方針

| 幅 | 方針 |
| --- | --- |
| mobile | 下部ナビ、1-2カラム、詳細は縦積み |
| tablet | 2-3カラム、地図とプレビューを併用 |
| desktop | 地図/詳細/一覧を広く使い、カードは密度高め |

## 禁止事項

- 明るい旅行サイト風の配色
- 白背景を主体にしたレイアウト
- 画像アイコン
- 過剰な説明テキストを画面に置くこと
- 装飾だけのカード乱用
- 視認性を壊す強すぎる発光
- 紫だけに偏った単調な画面

## 初期アセット方針

Phase3以降では、添付画像は直接コピーせず、配色・余白・カード感・発光感のみ参考にします。

実装では以下を使い分けます。

- DBの `image_url` がある場合: その画像を表示
- `image_url` がない場合: アプリ内のデフォルト幻想背景を表示
- `music_url` がある場合: BGMプレイヤーを有効化
- `music_url` がない場合: 「BGM準備中」と表示
